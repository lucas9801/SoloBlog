# Deploy To Cloudflare Pages

This project is ready for Cloudflare Pages.

## Recommended: Git Integration

1. Create a GitHub repository and push this project.
2. Open Cloudflare Dashboard.
3. Go to **Workers & Pages**.
4. Choose **Create application**.
5. Choose **Pages**.
6. Connect your GitHub repository.
7. Use these build settings:

```text
Framework preset: None
Build command: npm run build
Build output directory: dist
Root directory: /
Node version: 20
```

8. Deploy.

Cloudflare will also create a default preview URL in the project dashboard.

The current production domain is:

```text
https://blog.solus.games/
```

## Site URL

The build script uses URL values in this priority:

```text
SITE_URL
content/site.json baseUrl
```

`content/site.json` is already set to `https://blog.solus.games/`, so production builds use the custom domain by default.

If the domain changes later, update `content/site.json` or add this environment variable in Cloudflare Pages:

```text
SITE_URL=https://your-domain.com/
```

Then redeploy. This keeps canonical links, RSS, sitemap, and robots.txt correct.

## Optional: Direct Deploy With Wrangler

This is useful for quick manual publishing without Git integration.

```bash
npm run build
npx wrangler pages deploy dist --project-name soloblog-4w3
```

Or use:

```bash
npm run deploy:cloudflare
```

You need to log in to Cloudflare in the browser when Wrangler asks.

## Files For Cloudflare

- `wrangler.toml`: Pages project name and output directory for Wrangler.
- `.node-version`: Node 20 for compatible builds.
- `public/_headers`: security and cache headers copied into `dist`.
- `public/_redirects`: RSS aliases copied into `dist`.
- `dist/robots.txt`: generated by `npm run build`.
- `dist/rss.xml`: generated by `npm run build`.
- `dist/sitemap.xml`: generated by `npm run build`.

## Normal Publishing Workflow

1. Create or edit Markdown files in `content/posts`.
2. Set `status: published`.
3. Run:

```bash
npm run lint
npm run build
```

4. Commit and push.
5. Cloudflare Pages deploys automatically.
